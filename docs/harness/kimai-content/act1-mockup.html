왜 정보부터 골라야 하는지 보고서 업무로 확인해 보겠습니다.
1-2 · report gap · lecture only

목표만 있으면 보고서 모습은 흔들립니다.

"다음 주 경영회의용 제품 리뷰자료 보고서 준비해줘."
무슨 제품인가
누가 읽는가
무엇과 비교하는가
보고서 초안 제품 범위 독자 수준 비교 기준
이 빈칸을 이름 붙이면 고를 정보가 보입니다.
1-3 · blank taxonomy · lecture only

김아이가 추측하는 빈칸은 다섯 가지입니다.

대상 제품과 독자가 없으면 보고서 방향을 임의로 잡습니다.
기준 기간과 비교 기준이 없으면 숫자의 의미가 흔들립니다.
제출 조건 형식과 기한이 없으면 완료 여부를 판단하기 어렵습니다.
그렇다고 모든 자료를 다 주면 좋은 것도 아닙니다.
1-4 · information classes · lecture only

모든 자료를 다 주는 것이 좋은 선택은 아닙니다.

필수 정보없으면 김아이가 추측합니다.
상황부 도움있으면 좋지만 순서가 이른 정보입니다.
방해 정보지금 목표를 흐릴 수 있습니다.
정보 바구니 지금 꼭 필요 나중에 도움 지금은 치움
이 작업이 하네스 엔지니어링의 첫 번째 장치입니다.
1-5 · context curation · lecture only

정보 선별은 김아이의 데스크를 정리하는 일입니다.

올릴 자료이번 업무에 바로 필요합니다.
잠시 둘 자료도움은 되지만 지금은 선택입니다.
치울 자료현재 목표를 흐립니다.
Context Curation 데스크 위 자료 정리 업무 전 선별 추측 빈칸 줄이기
이제 같은 기준으로 첫 실습을 진행합니다.
1-6 · handoff to separate practice

실습에서는 문장을 쓰지 않고 정보를 고릅니다.

보고서 목표 읽기 무엇을 맡기는지 먼저 확인합니다.
정보 후보 고르기 필요한 정보와 과한 정보를 구분합니다.
빈칸 리포트 보기 김아이가 무엇을 추측할지 확인합니다.
첫 문제는 제품 리뷰자료 보고서입니다.
