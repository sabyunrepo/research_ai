# 김아이 워크숍 Act 1 강의 구성 요약

이 문서는 Act 1 `정보 선별`만 검토하기 위한 마크다운 원본이다. Act 2~6은 아직 확장하지 않는다. HTML 목업에서도 Act 1에만 실제 용어 연결 슬라이드를 추가한다.

## Act 1의 역할

Act 1은 김아이에게 일을 맡기기 전에 "무엇을 알려줄지"부터 고르는 구간이다. 제품 리뷰자료 보고서를 예시로 사용해, 목표만 던지면 김아이가 빈칸을 추측한다는 문제를 보여 준다.

하네스 엔지니어링 관점에서는 `Information Selection / Context Curation`의 출발점이다. 아직 지시문을 잘 쓰는 법이나 컨텍스트 파일 구조를 설명하지 않는다. 먼저 수강생이 필요한 정보, 있으면 도움 되는 정보, 이번 일에는 빼야 할 정보를 나누게 만든다.

## 기존 자료 기반 확인

현재 워크트리에서 직접 확인 가능한 `lecture-cuts/slide-spec.json`은 58장이다. 프로젝트 지침에는 원본 87장 기준이 남아 있으므로, 현재 파일은 원본 전체가 아니라 압축 또는 재생성된 인벤토리일 수 있다.

Act 1에는 다음 원본 개념을 압축해 사용한다.

| 원본 슬라이드 | Act 1에서 쓰는 방식 |
| --- | --- |
| `02-failure-patterns` | 실패를 성능 문제가 아니라 절차와 입력 문제로 설명 |
| `02-1-why-llms-fail` | 김아이가 빈칸을 추측으로 채운다는 핵심 원리 |
| `02-2-failure-example-read-before-edit` | 정보 없이 시키면 없는 내용을 상상한다는 사례 |
| `02-3-failure-example-skip-test` | 완료 기준이 없으면 성공처럼 보일 수 있다는 사례 |
| `02-4-failure-example-context-drift` | 오래된 가정이 현재 작업을 오염시킨다는 사례 |
| `02-5-improvement-process-guardrails` | 과한 문장보다 절차와 기준이 필요하다는 브릿지 |

## 슬라이드 흐름

1. 프롬프트보다 자료를 먼저 골라야 합니다.
2. 목표만 있으면 보고서 방향이 흔들립니다.
3. 김아이가 추측하는 빈칸은 다섯 가지입니다.
4. 자료는 많을수록 좋은 것이 아닙니다.
5. 데스크는 김아이가 일할 자료만 올려두는 공간입니다.
6. 관련 없는 자료는 김아이의 판단을 흐립니다.
7. AI의 데스크가 바로 컨텍스트입니다.
8. 자료가 부족하면 김아이는 빈칸을 직접 채웁니다.
9. 첫 실습은 김아이에게 줄 자료를 고르는 일입니다.

## 설명 구조

Act 1은 단순히 `데스크 정리 = Context Curation`이라고 말하지 않는다. 다음 순서로 설명한다.

1. 현실 비유: 데스크는 김아이가 일을 하기 위한 자료만 올려놓는 공간이다.
2. 작동 원리: 필요한 자료가 가까이 있으면 판단이 빨라지고, 불필요한 자료가 섞이면 판단이 흔들린다.
3. 문제가 생기는 조건: 해당 일과 관련 없는 자료가 데스크에 올라와 있으면 의사결정에 혼란이 생기고 작업도 느려진다.
4. 실제 용어: AI에서 이 데스크는 대화나 세션이 진행되며 쌓이는 내용, 즉 `Context`다.
5. 실습 이유: 그래서 Act 1에서는 김아이에게 줄 컨텍스트를 먼저 고른다.

## 실제 용어 매핑

| 비유 | 실제 용어 | 하네스에서 하는 일 | 실습에서 확인할 증거 |
| --- | --- | --- | --- |
| 김아이 데스크 | Context | 대화/세션 안에서 김아이가 참고하는 자료와 지시가 쌓이는 공간이다. | 관련 없는 자료가 있으면 판단이 흔들리고 결과가 느려진다. |
| 데스크 정리 | Context Curation | 이번 작업에 필요한 입력 자료를 고르고, 방해되는 자료를 제외한다. | 빈칸 리포트에서 김아이가 추측할 항목이 줄어든다. |

Act 1에서 "정보를 고른다"는 말은 단순히 자료를 많이 모으는 일이 아니다. 하네스에서는 김아이가 이번 작업에서 볼 컨텍스트의 범위를 정하는 일이며, 이 범위가 이후 지시문과 결과물의 출발점이 된다.

## 원본 개념 반영

원본의 실패 사례는 Act 1에서 하나의 진단으로 묶는다. `읽지 않고 고치면 API를 상상한다`, `테스트를 느슨하게 하면 성공처럼 보인다`, `오래된 가정이 현재 작업을 오염시킨다`는 모두 김아이가 필요한 정보나 완료 기준을 받지 못했을 때 생기는 문제다.

따라서 Act 1은 정보 선별을 "자료 고르기"로만 말하지 않는다. 김아이가 추측할 빈칸을 미리 찾고, 어떤 빈칸은 다음 Act의 지시문, 컨텍스트, 검증 기준으로 넘겨야 하는지 구분하는 시작점으로 설명한다.

## 실습 연결

Act 1 설명이 끝나면 수강생은 지시문을 쓰지 않고 정보만 고른다. 첫 실습의 목표는 "제품 리뷰자료 보고서"를 만들기 전에 김아이의 데스크 위에 올릴 자료를 선별하는 것이다.
