최종 보고는 덱 품질의 증거입니다

무엇을 만들었는지보다 어떤 검증을 통과했는지가 다음 세션의 기준이 됩니다.

  • 변경 파일
  • 실행 명령
  • 검증 결과
  • 남은 위험과 확인 URL
발표 포인트최종 보고는 “완료했습니다”가 아니라 다음 Claude와 발표자가 믿을 수 있는 증거 목록이어야 합니다.
변경:
- slide-spec.json
- slides/02-review-gate.html
검증:
- node scripts/verify-deck.js
결과:
- broken link 0
- note exposure 0
- overflow 0
확인:
- /deck.html
- /presenter-review.html
