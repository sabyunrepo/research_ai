Skill

스킬은 호출 조건, 절차, 자료를 분리합니다

SKILL.md는 가볍게 두고, 무거운 예시와 실행물은 폴더로 나눕니다.

  • description: 언제 자동 호출할지 정합니다.
  • 본문: 모델이 바로 따를 짧은 절차입니다.
  • references/: 긴 설명, 예시, 판단 기준표를 둡니다.
  • scripts/: 반복 실행할 검증/변환 코드를 둡니다.
  • assets/: 템플릿, 이미지, 샘플 파일을 둡니다.
발표 포인트스킬은 호출 조건, 실행 절차, 참고 자료, 실행 코드, 재사용 파일을 분리해야 가볍게 유지됩니다.
