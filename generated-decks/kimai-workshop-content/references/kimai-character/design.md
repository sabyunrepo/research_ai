# KimAI Character Design

이 폴더는 김아이 이미지를 새로 만들 때 기준으로 삼는 프로젝트 로컬 레퍼런스입니다. 새 슬라이드 이미지를 만들기 전에 이 문서와 레퍼런스 PNG를 먼저 확인합니다.

## Character Identity

- 이름: 김아이
- 역할: 사무실에 새로 온 AI 신입사원, 강의 안에서는 학습자가 일을 맡기는 대상
- 인상: 친근하고 성실한 주니어 직원, 과장된 로봇이나 미래형 캐릭터가 아님
- 체형: 큰 둥근 머리와 작은 몸, 단순한 손, 귀엽지만 업무용 슬라이드에 맞는 절제된 표정
- 얼굴: 둥근 얼굴, 큰 검은 타원형 눈, 작은 입, 옅은 파란 볼
- 머리: 흰색 짧은 머리, 앞머리가 둥글게 내려오고 위쪽에 한 가닥 솟은 컬

## Required Visual Features

- 오른쪽 귀 쪽에 둥근 헤드셋 이어컵
- 헤드셋 위로 가는 안테나와 작은 원형 팁
- 흰 셔츠
- 밝은 파란 넥타이
- 작은 직사각형 명찰
- 파란 포인트는 `#2563eb` 계열만 사용
- 검은 손그림 라인, 둥글고 굵은 외곽선
- 흰 배경 또는 거의 흰 배경

## Accessories

- 노트북: 전면에 단순한 파란 로봇 얼굴 아이콘을 넣을 수 있음
- 책상: 얇은 검은 선과 흰 면으로 구성
- 문서: 체크리스트, 자료 묶음, 클립보드, 빈 문서 보드
- 작은 소품: 식물, 펜컵, 머그컵은 가능하지만 장면의 중심이 되면 안 됨

## Style Rules

- 검은 라인 + 흰 면 + 파란 포인트만 사용합니다.
- 그림자는 쓰지 않거나 아주 옅은 파란 바닥 그림자만 사용합니다.
- 그라데이션, 광택, 3D 렌더, 복잡한 배경은 쓰지 않습니다.
- 한 슬라이드 안에서 김아이 얼굴 비율과 헤드셋 위치가 흔들리면 실패입니다.
- 레퍼런스 이미지는 형태 기준입니다. 이미지 안의 글자나 마크는 정확한 텍스트 기준으로 쓰지 않습니다.

## Text Rule

이미지 생성 모델은 한글과 영문을 틀릴 수 있습니다. 슬라이드에서 정확해야 하는 텍스트는 다음 중 하나로 처리합니다.

- HTML/CSS 텍스트로 별도 렌더링
- 로컬 합성 레이어에서 정확한 폰트로 삽입
- 이미지 안에는 빈 카드, 빈 보드, 간단한 선만 남김

텍스트를 이미지 안에 반드시 넣어야 할 때도, 생성 프롬프트에는 넓은 빈 공간을 요구하고 최종 텍스트는 후처리로 넣습니다.

## Do Not Use

- 금속 로봇, 휴머노이드 로봇, SF 배경
- 어두운 테마, 네온, 과한 파란색 면적
- 일반 회사원 캐릭터처럼 헤드셋/안테나가 빠진 모습
- 귀가 동물처럼 보이는 형태
- 복잡한 UI 화면, 코드 화면, 실존 회사 로고
- 작은 글씨가 많은 문서
- 손가락, 안테나, 캐릭터 일부가 잘린 크롭

## Slide Composition Guidance

- 김아이는 장면의 중심 피사체 또는 의미를 연결하는 안내자로 둡니다.
- 용어 지도나 목차 이미지는 김아이보다 카드가 너무 많아지면 품질이 떨어집니다. 카드가 4개를 넘으면 이미지 내부 텍스트를 줄이고, 정확한 용어는 슬라이드 왼쪽 앵커나 별도 텍스트로 처리합니다.
- 기존 고품질 자산처럼 한 이미지에는 1개 장면, 1개 핵심 행동, 1-3개 큰 소품을 우선합니다.

## Quality Checklist

- 김아이의 머리 모양, 헤드셋, 안테나, 파란 넥타이가 모두 보이는가?
- 검은 선과 파란 포인트가 기존 덱 이미지와 비슷한가?
- 캐릭터가 과장된 로봇이나 일반 사람으로 바뀌지 않았는가?
- 슬라이드에 들어갔을 때 얼굴과 소품이 충분히 크게 보이는가?
- 이미지 안의 텍스트를 그대로 믿지 않도록 후처리 계획이 있는가?
