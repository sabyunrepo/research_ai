# Presentation Design: Hand-drawn Minimal

## Visual Direction
- 제목은 약간 손글씨 느낌의 폰트를 사용한다.
- 낙서 느낌의 단순한 일러스트를 보조 그래픽으로 사용한다.
- 그라데이션, 광택, 과한 그림자, 장식 효과는 사용하지 않는다.
- 색상은 기본 흑백 + 포인트 1색, 최대 2색까지만 사용한다.
- 레이아웃은 본질만 남기고, 요소 수를 최소화한다.

## Typography
- Headline: hand-drawn 또는 handwritten 느낌의 폰트. 슬라이드의 큰 결론 문장, 카드의 핵심 라벨, 손그림 도식 안의 큰 단어에만 사용한다.
- Body: 읽기 쉬운 sans-serif. 본문, 설명문, presenter/review 메타 정보, 작은 보조 문구에 사용한다.
- Code: monospace. 코드, 파일 경로, 명령어, `목표:`, `검증:`처럼 업무 문서 형식을 그대로 보여주는 창 안의 행에만 사용한다.
- Label: Body와 같은 sans-serif를 쓰되 굵게. eyebrow, 템플릿 메타, 작은 보조 라벨에 사용한다.
- 제목은 크고 느슨하게, 본문은 짧게 유지한다.
- 한 화면 안에서 Headline, Body, Code, Label 역할을 섞지 않는다. 같은 역할의 글자는 같은 폰트 토큰을 사용한다.

## Graphic Rules
- 일러스트는 선 기반 낙서 스타일로 제한한다.
- 사진, 3D 이미지, 복잡한 아이콘 세트는 피한다.
- 장식 그래픽은 개념 이해를 돕는 경우에만 사용한다.

## Color Rules
- background: #ffffff
- text: #111111
- accent: #2563eb
- secondary accent: optional
- gradient: 금지
- glow / glass / blur: 금지

## Layout Rules
- 한 화면에 하나의 핵심 메시지만 둔다.
- 카드, 박스, 테두리는 꼭 필요할 때만 사용한다.
- 여백을 크게 두고, 요소 정렬은 단순하게 유지한다.
- 표나 그래프보다 손그림 도식, 짧은 문장, 큰 빈 공간을 우선한다.

## Page Patterns
- hero: 손글씨 제목 + 작은 낙서 일러스트 + 짧은 부제
- concept: 한 문장 정의 + 선 기반 비유 그림
- process: 3단계 이하 손그림 흐름도

## Tokens
- accent: #2563eb
- background: #ffffff
- surface: #ffffff
- mutedSurface: #f5f7fb
- radius: 10
- density: comfortable
- fontDisplay: "Comic Sans MS", "Marker Felt", "Bradley Hand", "Noto Sans KR", sans-serif
- fontBody: "Noto Sans KR", system sans-serif
- fontCode: SFMono-Regular, Menlo, Consolas, "Noto Sans Mono", monospace
- fontLabel: fontBody, 800-900 weight

## Applied Rules For This Deck
- 모든 컷은 생성 이미지 기반 손그림 일러스트를 사용한다.
- HTML/CSS로 직접 도형, 그래프, SVG 다이어그램을 만들지 않는다.
- 한 컷의 본문 bullet은 3개 이하를 기본값으로 한다.
- 코드 예시는 필요한 컷에만 짧게 넣는다.
- 최신 기법은 Prompt, Context, Skill, Agent, Hook, Tool, Evaluation, Loop 순서로 배치한다.
